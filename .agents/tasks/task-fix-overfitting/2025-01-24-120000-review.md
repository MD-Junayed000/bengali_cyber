# Anti-overfitting regularization sweep for Bengali cyberbullying classifiers

Both notebooks received a coordinated set of regularization techniques aimed at closing the train-val F1 gap. Track 1 (lightweight CNN+BiGRU) adds mixup, R-Drop KL consistency loss, stronger weight decay, tighter gradient clipping, reduced learning rates post-unfreeze, a cosine annealing schedule, earlier SWA start, doubled augmentation copies, and reduced patience. Track 2 (BanglaBERT transformer) adds layer-wise learning rate decay (LLRD), bottom-4 layer freezing, multi-sample dropout, hidden dropout with LayerNorm on the pooled representation, R-Drop, cosine-with-warmup scheduling, label smoothing, and stricter gradient clipping.

Watch for: R-Drop doubles forward-pass compute in both notebooks, effectively halving throughput — no output evidence that final wall-clock or memory stayed within Kaggle T4x2 limits (**likely**). The lightweight notebook still shows a +5pp train-val gap at early stopping (epoch 22), suggesting the regularization battery reduced severity but did not eliminate the overfitting pattern (**confirmed**). Track 2's validation loss diverges monotonically from epoch 3 onwards despite F1 improvements, which hints that threshold tuning is papering over a miscalibrated model (**confirmed**).

## High-level view

The lightweight model's overfitting was attacked from multiple angles simultaneously: the peak LR was cut 37% (8e-4 → 5e-4), post-unfreeze body LR halved (0.30 → 0.15), weight decay tripled (1e-3 → 3e-3), grad clip halved (1.0 → 0.5), dropout raised (0.50 → 0.55), label smoothing doubled (0.05 → 0.10), epochs cut (35 → 25), patience shortened (8 → 5), augmentation doubled (1 → 2 copies), and two new losses (mixup α=0.3, R-Drop α=0.7) were injected into the training loop. The result cut the gap from +10pp at early stop to +5pp, but the model still peaks at the same test Macro-F1 (0.78), and there's no output showing the new configuration running to completion with the ensemble.

Track 2 replaced the simple linear-warmup-to-zero schedule with cosine-warmup, added LLRD (factor 0.9) so lower layers train more slowly, froze the bottom 4 encoder layers entirely, and wrapped the classifier in multi-sample dropout (5 samples) with an extra LayerNorm and hidden-dropout layer before pooling. R-Drop was added identically to Track 1. The training log shows val loss diverging from 0.0720 at epoch 2 to 0.1483 at epoch 7 while F1 simultaneously rose from 0.80 to 0.83 — the model confidently miscalibrates on the samples it gets wrong. The final test Macro-F1 (0.8365) is solid, but the underlying calibration concern means probability thresholds may not transfer to new data.

Both notebooks share an R-Drop implementation but differ in how they compute symmetric KL (Track 1 uses element-wise Bernoulli KL including the (1-p) complement; Track 2 uses `F.kl_div` which only accounts for positive probability mass). This means Track 2's R-Drop penalty is approximately half as strong as Track 1's at the same alpha.

## Hyperparameter tightening in Track 1

Fourteen parameters were changed simultaneously. The most impactful cluster is the LR+weight-decay reduction: LR from 8e-4 to 5e-4, body-unfreeze factor from 0.30 to 0.15, embed-unfreeze from 0.05 to 0.02, and weight decay tripled. The post-unfreeze schedule was changed from a linear warmup/decay to cosine annealing (via a lambda that uses `cos(pi * s/remaining)`).

The gap column in the training log: it goes negative during frozen phase (underfitting), crosses zero at epoch 9, and reaches +5pp by epoch 22 when early stopping fires. The old baseline hit +10pp, so this is a ~50% reduction in overfitting severity. However, the absolute best val F1 is still 0.7807 — identical to the old SWA-selected checkpoint — meaning the regularization didn't unlock generalization, it merely slowed memorization.

The `NUM_DROPOUT_SAMPLES` increase from 5 to 8 combined with R-Drop's doubled forward pass means effective compute is roughly 3.2x per training step. The 19s/epoch wall-clock in the log seems too close to the original 19s — this might indicate the notebook wasn't actually re-run with these changes, or the R-Drop/multi-dropout paths weren't active during that particular cell execution.

## Mixup and R-Drop in Track 1

The mixup implementation is label-only (no input interpolation): it samples λ from Beta(0.3, 0.3) and blends the target labels from shuffled pairs. The blended labels are passed to the focal loss, so the effective label smoothing is now the sum of `LABEL_SMOOTHING=0.10` plus mixup's implicit smoothing — potentially over-smoothing small classes like `threat` (14% of samples).

R-Drop's combined loss is `(L1+L2)/2 + 0.7 * KL`. The alpha of 0.7 is substantially higher than the original R-Drop paper's typical range (0.1–0.5). At 0.7 with an already strongly-regularized model (weight decay 3x, label smoothing 0.10, mixup 0.3, dropout 0.55), there's a risk the KL term dominates and suppresses learning — which could explain why val F1 peaked at the same 0.7807 as before despite all the new machinery.

## Multi-sample dropout train/eval asymmetry in Track 2

At training time, the classifier receives logits averaged over 5 dropout-masked passes. At inference, only a single pass is used. This asymmetry means the model optimizes for smoother logits but is evaluated on noisier single-pass outputs. This could partly explain the val-loss divergence: the eval-time loss is computed on single-pass logits that are systematically different from what the model was trained to produce.

## Incomplete Bernoulli KL in Track 2's R-Drop

```python
def symmetric_kl_divergence(p, q):
    kl_pq = F.kl_div(q.log(), p, reduction='batchmean')
    kl_qp = F.kl_div(p.log(), q, reduction='batchmean')
    return (kl_pq + kl_qp) / 2
```

These are sigmoid probabilities (independent Bernoulli per class), not a categorical distribution. `F.kl_div` computes `sum(target * (target.log() - input))`, which gives `sum(p * log(p/q))`. For Bernoulli random variables, the full KL is `p*log(p/q) + (1-p)*log((1-p)/(1-q))`. The second term is missing. Track 1's manual implementation correctly includes both terms. At RDROP_ALPHA=1.0, Track 2 is applying roughly half the intended consistency penalty.

## Validation loss divergence in Track 2

```
Epoch 2: VaLoss 0.0720
Epoch 7: VaLoss 0.1483  (+106%)
```

F1 went from 0.8026 to 0.8271 over the same period. The model is becoming more confident on correct predictions (pushing sigmoid outputs further from 0.5) while simultaneously becoming more wrong on its mistakes. The multi-sample dropout train/eval asymmetry likely contributes: focal loss at gamma=2 sees averaged logits during training but the eval loss is computed on single-pass logits that carry more extreme confidence.

The final test Macro-F1 of 0.8365 may not be robust: if this model is deployed with fixed thresholds tuned on validation, domain shift will hit it harder than a well-calibrated model.
